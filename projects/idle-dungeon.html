idle-dungeon

Playable · Vanilla JS

A browser idle/incremental dungeon crawler — no framework, no build step, just HTML, CSS, and JS. Start a run, let it play itself, and come back stronger after every death.

Core loop

  • Start a run and the hero descends automatically, one tick every half-second (1x/2x/4x speed, your choice). There's nothing to click mid-run except pause — the game plays itself.
  • Each floor is rolled on the spot: a normal fight, an elite fight, a treasure room (straight gold), or a shrine (full heal). Combat itself is automatic — hero and enemy trade hits every tick based on ATK/DEF, with a small crit chance from your Fortune stat.
  • XP, levels, and loot happen mid-run — level-ups heal and buff the hero on the spot, and defeated enemies have a chance to drop temporary weapon/armor/vitality bonuses that only last for the current run.
  • Death ends the run and converts how deep you got plus your hero level into essence — the permanent currency.
  • Essence buys permanent upgrades (Vitality, Strength, Toughness, Fortune) in the Essence Shop between runs, so every death makes the next run start a little stronger.

That's the whole prestige loop: run, die, cash out, upgrade, go deeper next time. Essence, upgrade levels, and best-floor progress are saved automatically to your browser's local storage; a run in progress is not preserved across a refresh, but everything permanent is.

As with any client-side browser game, the full JS source is visible to anyone via your browser's view-source or dev tools once the page loads — that's true regardless of whether the project's home repository is public or private, since the code has to ship to your browser to run. Not a bug in this build, just how client-side web games work.
