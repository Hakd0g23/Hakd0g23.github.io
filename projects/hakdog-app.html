hakdog-app

Live demo · React Native / Expo

A mobile-first budgeting app for splitting one shared pool of money across three spending envelopes — Needs, Wants, and Invest — plus a separate running Travel fund. Built with React Native and Expo, with no backend at all: every number lives in local storage on the device (or in the browser's local storage, for the web build below). There's no account to create and nothing ever leaves the device.

Open the live demo →

What it does

  • Onboarding, not a login — first launch just asks for a name and an avatar. No sign-in, no email, no password.
  • Set a monthly income and it auto-allocates across the four envelopes using a default split, which is fully editable in Settings.
  • Log transactions against an envelope and watch its remaining balance and progress bar update immediately.
  • A dedicated Travel tab tracks a running savings fund separately from the monthly spending envelopes, since travel money isn't meant to reset every month the way the others do.

Try the demo

The live demo above is the actual web export of the app, not a mockup — it opens on the same first-launch onboarding screen a fresh install would show, so expect to pick a name and an avatar before you see the dashboard. Everything you enter is saved only in that browser's local storage, so it's safe to explore with your own numbers: nothing is transmitted anywhere, and clearing the site's storage (or opening a private window) resets it back to onboarding.

The screenshot above is seed data from a fresh demo session, not a real budget. If you're evaluating the live demo yourself, the same applies — treat it as scratch space, not somewhere to park real financial figures, since it's a public URL running entirely client-side.
